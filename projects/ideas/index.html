Random Ideas Coalescing Into a Star

Dec 4th, 2013

Stuff in my mind

So in my version of computing a voxelwise connectivity map, I pick an ROI then compute the connectivity with the rest of the brain. In order to get a good baseline, I apply a wavelet permutation of the time-series and then redo the connectivity map. I apply this procedure say 500 times and get 500 permutation connectivities at each voxel. This way, I can determine the p-value. I might want to compare this approach with the simpler approach of thresholding the connectivity map based on an r value with a p-value related to something sensible like 0.05, 0.01, or 0.001. If there are any negative connections, I’d prefer to threshold those out.

Ok so the result of the above step is a thresholded positive connectivity map. I want to then visualize this map across all the participants, and then measure the variation across participants. So what kinds of things change:

  • clusters
  • are all the clusters within known group-level or network-level connectivity?
  • are there some clusters more prominent and consistently prominent in certain subjects?
  • when a connection is missing in a subject, how does it manifest?
  • we can see variation in the amplitude of the connections. there can be a complete here and not here effect. there can be spatial differences in the location.

Interested in this approach. What does applying this to the time-series tell us about the time-series?

Power Analysis

Some thoughts about doing various types of analyses here with the aim. For instance, one approach would be to get clusters of frequencies based on the spatial maps. Other thing that could be done is that I could bandpass a small window and then compute the connectivity at that window (would need to figure out

Information is present at different frequency bands. These might reflect different highways of information processing, potentially occurring in parallel. One question is how the total correlation between two items is influenced by the partial bandwidth correlation.

One thing is that if I’m clustering each frequency across the entire brain with other frequencies, it’s possible that some voxels will be related across a window of frequencies but other’s won’t. So fucking suddenly, I have this massively complex problem.

Going back, what does this tell us about the brain or something interesting.

  1. A clinical group could show abnormalities in the power spectrum. If a region is having some problems in it’s functioning then it’s time-series might be a bit screwy and this would reflect in the power-spectrum.
  2. Certain brain regions might have sensitivity for a particular power-spectrum (e.g., frontal polar cortex for lower frequencies).

Get the cluster for each window. Then look at the difference between groups based on that window either via multivariate or a combination of univariate (e.g., number of significant t-tests in that window). This done at each voxel and the results could be plotted voxelwise.

CWAS using GLM

Get the GLM map at each voxel looking at brain-behavior relationship. For one voxel, we can then get a summary measure such as

  • the sum/mean of the t-values
  • the number of significant t-values (corrected…likely via clusters)
  • the number of significant clusters?
  • some summary of the cluster sizes? for instance, number of sig voxels divided by the number of clusters.

Regions found with the above approach would now literally be useful for subsequent seed-based analysis. And I think really the best way to determine the significance is via permutation test unless we can come up with a principled way to summarize. Also permutation is fairly easy to implement albeit time consuming. The other idea is that one could do a parcellation-based approach initially as sort-of a survey. The idea for all of this is that you could then use these regions to select subsequent regions of interest.

Ask phil if there is some literature about combining t-values and having something about combining their results. Oh I guess I could also do some kind-of binomial test? Or for now also could do a permutation test using one of the parcellations? I forgot, how long does the permutation of the GLM maps take? Some of the timing can be shortened by having the hat matrix computed in advance.
